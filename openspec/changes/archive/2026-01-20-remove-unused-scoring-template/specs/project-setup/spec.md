## MODIFIED Requirements
### Requirement: Project Structure
The project SHALL be organized according to the architecture specification with clear separation between application code, features, integrations, and tests. The project structure SHALL contain only actual feature directories and files in use, with no unused template artifacts or placeholder directories.

#### Scenario: Backend directory structure exists
- **WHEN** examining the project root
- **THEN** backend directory contains src/, tests/, scripts/ subdirectories (docker/ is at root level, openapi/ is auto-generated by FastAPI)

#### Scenario: Feature modules are organized
- **WHEN** examining the src/features directory
- **THEN** each feature (groups, subgroups, leaders, members, invites, contests, training, exports) has its own directory with models.py, schemas.py, repository.py, service.py, and policies.py files

#### Scenario: No unused template directories exist
- **WHEN** examining the project structure
- **THEN** no template directories (e.g., directories with placeholder names like `{scoring}`) exist in the codebase

#### Scenario: No duplicate directories exist
- **WHEN** examining the project structure
- **THEN** no duplicate directories exist (e.g., `backend/docker/` when `docker/` already exists at root level)

#### Scenario: No manually maintained OpenAPI files exist
- **WHEN** examining the project structure
- **THEN** no manually maintained OpenAPI YAML files exist (FastAPI auto-generates OpenAPI schema from code at `/docs` and `/openapi.json`)

#### Scenario: Documentation is consolidated
- **WHEN** examining the backend directory
- **THEN** documentation is consolidated into `DOCUMENTATION.md` instead of multiple separate files (DOCKER.md, DOCKER_MIGRATIONS.md, MIGRATIONS.md, DEVELOPMENT.md)
